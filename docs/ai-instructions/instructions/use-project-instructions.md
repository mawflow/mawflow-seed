# 查找并执行项目指令

## 元信息

- ID：TINST-001
- 类型：项目指令
- 状态：启用
- 维护位置：`docs/ai-instructions/README.md`
- 触发词：#项目指令、#关键字、#+指令、#T001、#P001、#T001/项目指令、#T指令号/关键字、#P指令号/关键字、项目指令、按指令、走某某流程、执行某某指令、精准命中、某个已登记指令名称、习惯用语、口头指令、项目黑话

## 目标

当用户在 Codex 会话中使用项目内短语、别名、专有名词或流程名称时，AI 应能从 `docs/ai-instructions/README.md` 定位到对应条目，读取完整说明后执行，避免只凭通用经验猜测。

## 适用范围

- 用户明确要求按某个项目指令执行。
- 用户使用 `#关键字`、`#+指令`、`#关键字：任务补充`、`#关键字/任务补充`、`#T001`、`#P001` 或带参数的精确调用来调用项目指令。
- 用户消息命中本库登记的触发词、别名、术语或经验主题。
- 当前任务需要判断项目内某个专有名词、习惯说法或历史经验。

## 输入形式

- 推荐格式：`#关键字` 或 `#关键字：任务补充`，例如 `#模版升级：按源模板最新提交计算漂移并执行`。
- `#+指令` 是“# + 指令关键字”的泛称：用户写 `# 关键字`、`#关键字`、`#关键字：补充` 或 `#关键字/补充` 都要进入指令匹配。
- 模板内置指令精确格式：`#T001` 或 `#T001/<关键字或任务补充>`，例如 `#T011/模版升级`。
- 业务项目特有指令精确格式：`#P001` 或 `#P001/<关键字或任务补充>`，例如 `#P001/客户发布`。
- 兼容格式：自然语言触发词、习惯用语、口头缩写，或完整 ID 格式 `#TINST-001/...`、`#PINST-001/...`。
- 习惯用语例子：“提主”“合主”“跑包/跑任务包”“交接任务”“客入/客主/客出/客户合主”“上线/发布”“同步镜像”“收口”“查历史坑”“模板漂移”“生成交付文档”。
- 如果 `#关键字`、`#+指令`、习惯用语或自然语言触发词命中多条指令、关键词含义不清、当前仓库角色无法判断，或精确调用编号和 `/` 后关键词含义冲突，必须先向用户确认，不得自行猜测。

## 执行步骤

1. 读取 `docs/ai-instructions/README.md`，查看指令、术语和经验列表；必要时查看根目录 `PROJECT_COMMANDS.md` 的人类快捷调用表。
2. 如果用户消息包含 `#T001` 或 `#T001/` 这类模板精确调用，先映射为模板内置 ID `TINST-001` 并精确定位指令；如果编号不存在，说明未找到该模板指令，并再按 `/` 后关键词尝试匹配。
3. 如果用户消息包含 `#P001` 或 `#P001/` 这类业务项目精确调用，先映射为业务项目 ID `PINST-001` 并精确定位指令；如果编号不存在，说明未找到该业务项目指令，并再按 `/` 后关键词尝试匹配。
4. 如果用户消息包含完整 ID 格式 `#TINST-001/` 或 `#PINST-001/`，兼容映射到对应内部 ID；后续输出和新增文档改用 `#关键字` 推荐格式，精确调用用 `#T001` 或 `#P001`。
5. 如果用户消息包含 `#关键字`、`#+指令`、`#关键字：任务补充`、`#关键字/任务补充`，先做轻量归一化：去掉 `#` 后空格，统一全角/半角冒号，把 `模板/模版` 等稳定同义写法合并判断，再匹配条目的名称、推荐调用、触发词、别名和适用场景。
6. 如果用户消息没有 `#`，但包含习惯用语、口头缩写或项目黑话，也按候选指令处理；同时查看 `experience-index.md`、`keyword-candidates.md`、`experience-candidates.md` 和 `execution-lesson-candidates.md` 中是否存在相关候选关键词。
7. `#关键字`、`#+指令`、习惯用语或自然语言触发词命中多个条目、关键词含义不清、当前仓库角色无法判断，或精确调用编号与 `/` 后关键词含义冲突时，必须先向用户确认要执行哪条指令；不得自行猜测。
8. 如果任务涉及实现、修 bug、测试、构建、发布、同步、执行脚本或历史踩坑，继续检索 `docs/ai-instructions/experience-index.md`；命中后按索引读取对应详情，不主动全量读取 `solutions/**`。
9. 命中单一条目时，读取该条目的完整说明文档，并按其中步骤执行。
10. 未命中任何条目时，按仓库通用规则处理任务；不要伪造不存在的项目指令。
11. 执行时仍需遵守 `.maw/codex-context.md`、`docs/ai-coding/`、安全脱敏、测试和提交规则。
12. 如果任务过程中发现可复用的新指令、术语或经验，且用户要求沉淀，按 `TINST-002` 更新本目录。
13. 如果任务提示词中出现高频项目关键词或影响边界的特有名词，按 `TINST-004` 更新候选台账或正式术语。
14. 如果执行中经历错误尝试后找到正确方法，且可能复用，按 `TINST-004` 更新执行经验候选。
15. 不得为了“了解经验库”主动全量读取 `docs/ai-instructions/solutions/**`；只有 `experience-index.md`、候选台账、用户明确路径或命中条目指向具体详情文件时，才读取该单个详情文件。

## 验证方式

- 确认使用的条目路径真实存在。
- 使用 `#关键字`、`#T001`、`#P001` 或带参数精确调用时，确认推荐调用、短编号、内部 ID、`docs/ai-instructions/README.md` 条目和 `PROJECT_COMMANDS.md` 快捷调用没有明显冲突；有歧义时已向用户确认。
- 确认最终行为没有越过条目中写明的适用范围和禁区。
- 如更新了代码、配置、文档或脚本，按项目通用规则完成必要验证，并在最终说明中写清楚使用了哪个项目指令以及 `experience_lookup` 结果。

## 冲突与覆盖规则

- 用户最新明确要求优先于本指令库旧记录。
- `docs/ai-coding/` 中的强制编码边界和脱敏要求仍然有效。
- 指令文档之间冲突时，以更具体、更贴近当前任务范围的文档为准，并把冲突记录到待更新项。
