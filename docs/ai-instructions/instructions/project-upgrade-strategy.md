# 指令：项目升级策略

## 元信息

- ID：TINST-023
- 类型：项目指令
- 状态：启用
- 维护位置：`docs/ai-instructions/instructions/project-upgrade-strategy.md`
- 触发词：#项目升级、项目升级、模板派生项目升级、升级到最新模板、同步模板新特性、源模板升级、upgrade decision matrix
- 适用范围：在模板派生项目中选择性同步新版模板能力，或把当前项目升级到指定源模板版本。

## 目标

`#项目升级` 不直接复制模板文件。AI 必须先解析源模板来源、盘点目标项目事实和源模板变化，生成升级取舍矩阵，再按风险分级增量执行。

## 源模板解析顺序

1. 用户输入。
2. `.local/.maw/template-source.yaml`。
3. `.maw/template-source.yaml`。
4. 当前仓库。

源模板读取优先级固定为：用户输入 > `.local/.maw/template-source.yaml` > `.maw/template-source.yaml` > 当前仓库。

`.maw/template-source.yaml` 只保存共享占位或 git 来源；个人本机路径只能写入 `.local/.maw/template-source.yaml` 或当次用户输入。

## 执行步骤

1. 确认升级类型：模板派生项目升级、模板化改造、单项模板能力升级或其它。
2. 读取 `.maw/upgrade-policy.yaml`、`.maw/template-source.yaml`、必要时读取 `.local/.maw/template-source.yaml`。
3. 盘点目标项目事实：README、`code/`、app_key、发布配置、仓库映射、secrets、`.local`、模块档案和项目私有规则。
4. 盘点源模板变化，只读取与本次升级相关的最小文件。
5. 使用 `docs/ai-instructions/templates/upgrade-decision-matrix.md` 生成取舍矩阵。
6. 按风险等级执行：
   - R0：文档说明补缺，可自动执行。
   - R1：AI 控制面新增字段或提示词增强，可自动执行但必须记录取舍。
   - R2：脚本、任务包、配置 schema，需更严格验证。
   - R3：发布、客户仓、密钥、环境策略，默认人工确认。
   - R4：`code/` 或业务行为变化，默认不由项目升级自动处理。
7. 验证后按中文人类优先格式收口，并记录 `upgrade_strategy_update`。
   - 本次升级产生实际改动并推送项目仓库后，必须运行目标项目的 `ops/scripts/sync-repository-mirror.sh plan` 或等价仓库级 mirror 计划命令，以有效计划判断是否同步 mirror；不要只凭原始 `repository_mirrors.enabled=false` 判断未启用。
8. 如果本次项目升级生成了可交给另一个目标项目会话执行的提示词、handoff prompt、任务包入口或执行说明，最终说明必须包含“目标项目使用方式”：
   - 先写清固定入口语：“复制以下提示词到目标仓库会话执行。”
   - 列出生成的提示词或执行入口文件，包含项目根相对路径和当前本机绝对路径；本机绝对路径只允许出现在当次最终说明里，不写入可提交文档正文。
   - 如果目标仓库 Codex 会话能访问当前机器上的本地文件，给出可直接复制到目标仓库会话的一行：

```text
请完成 <提示词或执行入口文件本机绝对路径> 任务
```

   - 如果目标仓库 Codex 会话不能访问该本机文件，说明：打开 `<提示词或执行入口文件项目根相对路径>`；若文件内有 `text` 代码块，复制代码块整段，粘贴到目标派生项目的 Codex 会话里执行；若是 `EXECUTE_PROMPT.md` 等无 `text` 代码块的入口文件，则复制文件全文。
   - 如果使用的是仓库内任务包目录而不是单个提示词文件，同时给出可复制的 `#跑任务包：<任务包目录>` 或 `执行任务提示词工程：<任务包目录>`；如果目标项目尚未包含该任务包，先按本次说明复制任务包目录再执行。

## 保护边界

升级必须保护：

- 根 README 和业务项目说明。
- `code/` 业务源码、业务运行配置和 app_key。
- `.local/` 真实资料和本机配置。
- secrets、repositories、releases、客户仓库映射和镜像目标。
- 现有模块档案、changelog、项目私有规则和发布流程。

## 验证方式

- 取舍矩阵已生成或说明无需生成的理由。
- 可提交文档中没有个人本机模板路径。
- 没有整文件覆盖目标项目 README、`code/`、secrets、repositories、releases、app_key 或模块档案。
- 运行目标项目可用的最小验证；若推送项目仓库，推送后以仓库级 mirror 计划命令的有效输出判断是否同步镜像。
- 如果生成了提示词、handoff prompt、任务包入口或执行说明，最终说明包含使用方式：能访问本机文件时用 `请完成 <本机绝对路径> 任务`；不能访问时打开文件，优先复制其中 `text` 代码块整段到目标项目 Codex 会话；入口文件没有 `text` 代码块时复制全文。

## 禁区

- 不把提示词、handoff prompt 或任务包入口文件的本机绝对路径写入迁移说明、任务包、提示词正文或其它可提交文档；本机绝对路径只在最终说明中作为当次使用方式给用户。

## 冲突与覆盖规则

- 用户最新明确要求优先。
- 目标项目当前事实优先于源模板默认值。
- 与 `TINST-009` 冲突时，本指令负责项目升级策略；`TINST-009` 只做内置任务包路由。
- 与 `TINST-024` 冲突时，本指令在目标项目执行升级；`TINST-024` 在模板仓库生成升级资产。
