# 更新项目指令与经验库

## 元信息

- ID：TINST-002
- 类型：项目指令
- 状态：启用
- 维护位置：`docs/ai-instructions/`
- 触发词：#新增指令、#T002、#T002/新增指令、新增指令、记录指令、更新指令、删除指令、维护 PROJECT_COMMANDS、记住这个术语、沉淀经验、总结经验、让 AI 记住

## 目标

当用户希望把项目内的新流程、专有名词、经验或偏好沉淀下来时，AI 应把它写入 `docs/ai-instructions/`，并更新总纲索引和根目录 `PROJECT_COMMANDS.md` 快捷查询入口，保证后续 Codex 会话能优先用 `#关键字` 速记调用；模板内置指令用 `#T001` 精确调用，业务项目特有指令用 `#P001` 精确调用。

## 适用范围

- 新增或更新项目内可执行流程。
- 新增、改名、废弃项目指令，或调整推荐关键字、内部指令 ID、命名空间、短编号、触发词、适用范围、推荐调用方式。
- 新增或更新业务术语、客户简称、模块别名、部署目标别名等专有名词。
- 从任务提示词或用户消息中沉淀反复出现的项目关键词，并逐步补齐名词定义和边界。
- 从 AI 执行过程中的错误尝试和正确方法中沉淀可复用执行经验。
- 新增或更新复盘经验、踩坑记录、偏好总结或固定处理方式。
- 废弃过时指令，并把状态和替代条目写清楚。

## 分类规则

- 可执行流程写入 `instructions/`。
- 名词解释写入 `terms/`。
- 尚未稳定的关键词先写入 `keyword-candidates.md`；多次出现并边界稳定后再升级为 `terms/`。
- 用户澄清、说明、纠偏和可复用边界信息先写入 `experience-candidates.md`；多次适用后再升级为 `lessons/` 或 `instructions/`。
- AI 试错后找到正确方法的执行经验先写入 `execution-lesson-candidates.md`；多次适用后再升级为 `lessons/`、`instructions/` 或编码规则。
- 复盘经验、踩坑记录和项目偏好写入 `lessons/`。
- 完整排查过程、命令序列、替代方案、风险分析或验收清单较长的解决方案写入 `solutions/<category>/<topic>.md`，`experience-index.md` 只保留概述、触发关键词、适用范围和详情链接。
- 正式经验、解决方案详情、执行指令或编码规则只要用于防止重复踩坑，都必须同步登记到 `experience-index.md`，让后续会话先命中索引再读取详情。
- 如果一个内容既是术语又有执行流程，优先写一份 `terms/` 解释，再在 `instructions/` 中引用它。

## 执行步骤

1. 先读取 `docs/ai-instructions/README.md`、`docs/ai-instructions/experience-index.md`、`docs/ai-instructions/keyword-candidates.md`、`docs/ai-instructions/experience-candidates.md` 和 `docs/ai-instructions/execution-lesson-candidates.md`，确认是否已有相同或相近条目。
2. 判断本次是新增、更新、合并、废弃还是改名。
3. 选择对应模板：`templates/instruction.md`、`templates/term.md`、`templates/lesson.md` 或 `solutions/_template.md`。
4. 使用英文小写短横线创建或更新文件名，例如 `test-environment-release.md`。
5. 在条目中写清楚触发词、适用范围、执行步骤、验证方式、禁区和冲突覆盖规则。
6. 为每个正式指令确定编号命名空间、推荐调用和精确调用：
   - 模板仓库维护的内置指令使用 `TINST-XXX`，精确调用为 `#TXXX` 或 `#TXXX/<消歧词>`。
   - 业务项目特有指令使用 `PINST-XXX`，精确调用为 `#PXXX` 或 `#PXXX/<消歧词>`。
   - 推荐调用格式为 `#最常用关键字`，优先选择极简、好记且不易误伤的关键词。
7. 更新 `docs/ai-instructions/README.md` 的对应列表，确保 ID、名称、触发词和链接一致。
8. 新增、改名、废弃指令，或调整推荐关键字、内部指令 ID、命名空间、短编号、触发词、适用范围时，必须同步更新根目录 `PROJECT_COMMANDS.md` 的推荐调用、精确调用、触发关键词、概要、可复制调用示例和完整说明链接。
9. 检查推荐关键字是否会命中多条指令；如果存在合理歧义，必须在 `PROJECT_COMMANDS.md` 写明精确调用，并在执行规则中要求先向用户确认。
10. 业务项目引入模板升级时，不得重排或占用已存在的 `PINST-XXX`；模板新增指令只追加到 `TINST-XXX` 命名空间。
11. 如果来自任务提示词的关键词仍处于粗略理解阶段，先记录候选词、出现次数、粗略定义、边界补充和相关模块。
12. 如果来自用户澄清、说明或纠偏，且有后续复用价值，先提取标题、关键词、内容摘要、适用范围和相关路径，写入经验候选。
13. 如果来自 AI 执行试错，且有后续复用价值，先记录错误尝试/症状、正确方法、触发场景、验证方式和相关路径/命令，写入执行经验候选。
14. 如果新增或升级正式经验、解决方案详情、指令或编码规则，更新 `experience-index.md`，确保 `EXP-XXX`、触发关键词、先做什么、详情链接和风险等级一致。
15. 如果详情写入 `solutions/**`，只在索引或候选台账中引用具体文件；不得要求后续 AI 主动全量读取 `solutions/**`。
16. 如果改动影响模板初始化或 Codex 启动阅读顺序，同步更新 `.maw/codex-context.md`、`TEMPLATE_OVERVIEW.md` 或 `docs/template-usage-guide.md`；根目录 `README.md` 属于业务项目，只在项目简介或人类入口确实变化时做最小合并。

## 编写要求

- 触发词要写用户真实可能说出口的短语，不只写抽象标题。
- 正式指令必须给出推荐调用和精确调用：推荐调用使用 `#关键字`；模板内置精确调用使用 `#T001` 或 `#T001/消歧词`，业务项目特有精确调用使用 `#P001` 或 `#P001/消歧词`。
- 执行步骤要面向 AI 行动，避免只写背景介绍。
- 对部署、同步、凭证、生产数据、客户隐私等高风险事项，必须写验证和禁区。
- 不写真实密钥、账号密码、生产连接串、客户隐私或不可外传内容。
- 不把单次聊天的临时安排误写成长期规则；临时规则要写明有效期或废弃条件。

## 验证方式

- `docs/ai-instructions/README.md` 中的链接能打开。
- `PROJECT_COMMANDS.md` 中存在对应推荐调用、精确调用和可复制调用示例，且调用格式、触发关键词、说明链接与总纲一致。
- 模板内置指令和业务项目特有指令没有混用编号命名空间；模板升级没有改写业务项目已有 `PINST-XXX` 编号。
- 推荐关键字没有明显误伤；如存在歧义，已写明必须先向用户确认。
- `experience-index.md` 中的正式经验链接能打开，且大型解决方案详情只保留在 `solutions/**`。
- 条目 ID 没有重复。
- 新增条目的触发词不会明显误伤其他流程。
- 废弃条目仍保留替代路径或废弃原因，避免后续 AI 误用。
- 没有新增会诱导 AI 主动全量读取 `solutions/**` 的规则。

## 冲突与覆盖规则

- 用户最新明确要求优先。
- 项目级指令只在当前仓库生效，不默认推广为用户全局偏好。
- 与 `.maw/`、`docs/ai-coding/`、发布或脱敏策略冲突时，先记录冲突并按更高风险、更保守的规则处理。
