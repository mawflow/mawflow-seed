# 解决方案详情库

本目录保存较大的具体解决方案内容，例如完整排查过程、命令序列、替代方案、风险分析、验收清单和复盘证据。它服务于“经验索引 + 方案详情”的两层结构。

## 读取边界

- AI 不得主动全量读取 `docs/ai-instructions/solutions/**`。
- 只有以下情况才读取具体详情文件：
  - `docs/ai-instructions/experience-index.md` 的某条 `EXP-XXX` 命中并链接到该文件。
  - `experience-candidates.md` 或 `execution-lesson-candidates.md` 的候选条目明确指向该文件。
  - 用户明确给出某个 `solutions/**` 文件路径。
  - 当前错误信息、命令或路径与某个索引条目精确匹配。
- 命中后只读取对应单个详情文件，不顺手读取同目录其它方案。

## 内容定位

- `experience-index.md`：写概述、关键词、症状、适用范围、风险等级、先做什么和详情链接。
- `solutions/**`：写完整解决方案正文。
- `lessons/**`：写较短的经验结论和判断方法。
- `instructions/**`：写必须执行的流程和检查清单。

## 建议分类

- `testing/`：测试命令、运行时、环境、用例发现问题。
- `release/`：发布、上线、回滚、验收、目标识别问题。
- `config/`：配置来源、profile 聚合、本机 overlay、密钥引用问题。
- `git/`：分支、远端、镜像、客户仓库同步问题。
- `docs/`：文档读取范围、模块档案、提示词工程和归档边界问题。

## 文档要求

每个方案详情文件应包含：

- 适用场景。
- 问题症状。
- 根因判断。
- 解决步骤。
- 验证方式。
- 反例和风险。
- 关联经验索引 ID。

不得包含真实密钥、token、账号密码、客户隐私、生产连接串、未脱敏日志或本机项目绝对路径。
