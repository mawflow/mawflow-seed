# 经验总结目录

本目录保存项目复盘经验、踩坑记录、偏好总结和长期最佳实践。经验文档不一定是可直接执行的流程，但应能帮助 AI 在后续任务中更快做出正确判断。

新增经验时请使用 `../templates/lesson.md`，并同步更新 `../README.md` 的“当前经验列表”。

## 目录结构

- 根目录：保存通用协作经验，例如指令库与提示词目录分工。
- `release/`：保存发布、部署、上线、回滚、验收等运维经验。这里的内容面向 fork 后的项目参考复用，默认不是所有项目必须照搬的强约束。
- 较大的具体解决方案、完整排查过程、命令序列和验收清单应放入 `../solutions/`，并在 `../experience-index.md` 中登记触发关键词和详情链接。

## 经验索引规则

- 新增或更新正式经验时，同步更新 `../experience-index.md`。
- AI 可以主动检索 `../experience-index.md`，但不主动读取大型详情文档。
- 如果经验正文已经变成大篇幅方案，应拆成一条简短 lesson 或索引摘要，并把完整方案迁移到 `../solutions/**`。

## 发布经验扩展规则

- 一个经验文件只聚焦一个技术栈、组件或发布目标，例如 `fastadmin-server-release.md`。
- 文档必须写清适用范围、不可直接复制的项目字段、可迁移模型、验收清单和反例风险。
- 如果某个 fork 项目需要把发布经验升级成强执行流程，应在该项目中复制或改写为 `instructions/` 下的项目指令。
