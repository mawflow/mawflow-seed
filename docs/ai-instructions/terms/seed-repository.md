## 定义

在本项目体系中，“种子仓库”和“模板仓库”统一指 `maw-project-template` 这个项目仓库。它是 MAW 新项目初始化、模板控制面演进、模板内置指令和可复用能力的源头。

从 `maw-project-template` 派生出去的业务项目是“派生项目”。派生项目应知道自己来自种子仓库，并尽量记录哪些能力来自种子仓库、哪些能力是项目自定义、哪些项目自定义能力适合回流到种子仓库。

Seed 来源通道用于区分同一套模板规则的读取来源：

- `内部来源通道`：内部受控派生项目优先从内部 `maw-project-template` 开发源读取升级资产。内部私有 Git URL 只能出现在受控配置、`.local/.maw/template-source.yaml` 或当次输入中，不写入公开提示词。
- `public_seed`：外部公开派生项目优先从公开 Seed 分发仓读取升级资产，当前公开仓口径为 `https://github.com/mawflow/mawflow-seed`。
- `unknown_legacy`：历史项目缺少来源通道或来源冲突时使用。AI 应输出检测证据并要求人工确认，不自动执行模板升级。

## 边界

- 包含：`.maw/` 控制配置、`docs/ai-instructions/` 指令与经验、`prompts/` 任务提示词、`ops/` 通用脚本、`.maw-template/` 升级资产，以及其它可复用模板能力。
- 不包含：派生项目私有业务代码、真实客户配置、项目私有 README、真实 secrets、`.local/` 本机内容、客户仓库专属规则和只适用于单个项目的临时补丁。
- 待确认：如果某项能力既可复用又依赖目标项目事实，先记录为“候选回流”，再由种子仓库维护者决定是否抽象成可选能力。

## 使用场景

- 用户说“种子仓库”“模板仓库”“源模板仓库”时，默认都按 `maw-project-template` 理解。
- 派生项目开发中发现某个项目自定义能力适合复用到更多项目时，应命中 `#种子仓库升级`。
- 派生项目执行 `#模板升级/#模版升级` 时，应先读取 `template_source.source_channel`；外部公开项目不得要求访问内部私有 Seed 源。
- 最终收口时必须判断本轮是否发现种子仓库可优化点，并在技术元数据中说明。

## 易混淆项

- `#模板升级/#模版升级`：在种子仓库中生成模板能力向派生项目传播的升级资产；在派生项目中计算模板漂移并执行升级。
- `#项目升级`：把种子仓库已有能力按取舍矩阵增量同步到当前派生项目。
- `#种子仓库升级`：把派生项目中发现的可复用优化或新增能力记录为回流建议，并生成一段在种子仓库执行的任务提示词。

## 注意事项

- 任何回流到种子仓库的建议必须默认向下兼容，只做增项或默认关闭能力，不要求历史派生项目立即改造。
- 记录回流建议时只能写项目根相对路径、脱敏事实和场景说明；本机路径、真实密钥、客户隐私和生产连接串不得写入可提交文件。
- 派生项目缺少能力来源字段时，不阻塞开发；按旧项目兼容口径推断并补充候选记录即可。
