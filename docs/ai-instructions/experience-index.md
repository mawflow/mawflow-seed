# 经验索引

本文件是 AI/Codex 可以主动检索的经验入口。它只记录经验概述、触发关键词、问题症状、适用范围、风险等级和详情链接，避免后续会话为了找一个坑而全量读取大型经验文档。

## 读取规则

- AI 可以在任务开始、执行命令前、报错后、收口前主动检索本文件。
- AI 可以同时检索 `experience-candidates.md` 和 `execution-lesson-candidates.md` 的表头与关键词。
- AI 不得主动全量读取 `solutions/**`；只有本索引、候选台账、用户明确路径或已命中经验条目指向具体详情时，才读取对应详情文档。
- 命中多个条目时，优先读取风险等级更高、适用范围更贴近当前任务的条目。
- 如果命中经验与当前代码、`.maw` 配置或用户最新说明冲突，以当前事实和用户最新说明为准，并更新索引或详情文档。

## 新增条目规则

- `ID` 使用 `EXP-XXX`，按新增顺序递增。
- `类型` 可填写 `candidate`、`execution_candidate`、`lesson`、`solution`、`instruction` 或 `coding_rule`。
- `详情文档` 可以指向 `experience-candidates.md`、`execution-lesson-candidates.md`、`lessons/**`、`solutions/**`、`instructions/**` 或 `docs/ai-coding/**`。
- `主动读取详情` 默认写 `no，先命中本索引`；只有短小且安全的说明文档可写 `yes`。
- 如果详情文档较长或包含完整解决方案，应放入 `solutions/**`，本索引只保留概述和触发条件。

## 收口要求

任务最终说明中的 `experience_lookup` 应说明：

- 是否检索了本索引。
- 命中了哪些 `EXP-XXX`。
- 是否读取了详情文档及原因。
- 是否新增或更新了经验索引、候选台账、正式经验、方案详情或项目指令。
